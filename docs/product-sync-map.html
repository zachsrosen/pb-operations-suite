3Cross-link IDs (how the four systems find each other)

After approval, every system gets stamped with the IDs of the other three so you can pivot from any record to its siblings. Only the wizard / approval engine writes these — the per-product Sync Modal does not.

5What's missing — the gap inventory

MissingSpec field coverage in externals

~30 of ~35 spec fields (efficiency, cellType, voc/isc/vmp/imp, tempCoefficient, phase, inverterType, mpptChannels, chemistry, mountType, gaugeSize, deviceType, connectivity, etc.) live in our DB only. None are on HubSpot, Zoho, or as discrete Zuper custom fields. Zuper gets a single concatenated specification string for 4 categories.

MissingSync log / audit trail

No CatalogSyncLog table exists. BOM has BomHubSpotPushLog; catalog sync has nothing. Sync Modal results disappear when closed. No ActivityLog entries are written by either sync path. For forensics, you can only inspect PendingCatalogPush rows + IDs on InternalProduct.

PartialZoho group_name coverage

Only MODULE and INVERTER are confirmed in zoho-taxonomy.ts. BATTERY, EV_CHARGER, RACKING, MONITORING, GATEWAY, OPTIMIZER, ELECTRICAL_BOS, TESLA_SYSTEM_COMPONENTS, RAPID_SHUTDOWN, D&R, SERVICE, ADDER_SERVICES, PROJECT_MILESTONES → items create in Zoho without a group. Logs a console warning, no UI surfacing.

MissingSync Modal skips cross-link writes

If you create a Zoho/Zuper/HubSpot record via the per-product Sync Modal (not the wizard), the cf_internal_product_id / internal_product_id / etc. cross-link fields are NEVER written. Only executeCatalogPushApproval stamps cross-links. Result: the new external record is orphaned from the other systems.

PartialPhoto upload is Zoho-only

HubSpot Product images and Zuper Product images are never written. Zoho upload itself is best-effort — failures append a warning to the outcome message but don't fail the approval.

PartialDimensions don't reach Zuper

length, width, weight flow to Internal + HubSpot + Zoho but are not on the Zuper sync payload. hardToProcure only reaches HubSpot.

MissingHubSpot manufacturer enum dead-end

manufacturer is a HubSpot enumeration. If the brand isn't a known enum value, the entire HubSpot push fails with a 400 — no fallback to "manufacturer_other" or auto-creation of a new enum option. The wizard accepts arbitrary brand text, so this can fail silently after the form succeeds.

RaceGuarded link-back can orphan duplicates

When the Sync Modal creates an external record, the link-back uses updateMany WHERE id=… AND xxxItemId IS NULL. If two admins sync simultaneously, the second push wins in the external system but the InternalProduct still points to the first ID — second external record becomes an orphan, only logged via console.

AsymmetryWizard auth vs. Sync Modal auth

Anyone authed can submit through the wizard (auto-approves and creates external records). Sync Modal restricts to ADMIN/EXECUTIVE only. The wizard creates more downstream state with weaker auth than the modal.

AdjacentStock not auto-created

InventoryStock and StockTransaction tables exist but no part of the create flow populates them. Stock lives in its own /api/inventory/stock path and starts at zero.

MissingQuickBooks / OpenSolar slots exist but unwired

The systems array supports a "QUICKBOOKS" value in code comments, and UpsertHubSpotProductInput accepts qboProductId + writes qbo_product_id on HubSpot — but nothing creates a QBO product. PendingCatalogPush.openSolarId field exists for the sales-request flow but no sync path uses it.

PartialCross-link writes are best-effort

If the Zoho/Zuper/HubSpot cross-link PATCH fails, the message is appended as (Warning: …) to the outcome but the overall status still flips to APPROVED. The Zoho cf_internal_product_id write on a freshly-created Zoho item swallows errors entirely with no warning.
